## 關鍵技術原則

### 基本面 Lookahead 防護（Q15-B）

**為什麼重要：** 2024 Q1 EPS 是 2024-05-15 才公布，回測時若 2024-01-01 就用 EPS=8 來判斷，等於用了「未來資訊」，績效嚴重失真。

**保守延後規則（按法定公布期限）：**

| 資料類型 | 延後天數 | 範例 |
|---------|:------:|------|
| 季 EPS | **+45 天** | Q1 結束 3/31 → 5/15 起可用 |
| 月營收 | **+10 天** | 1 月 → 2/10 起可用 |
| 年營收 | **+90 天** | 年底 12/31 → 隔年 3/31 起可用 |
| TTM EPS | 跟隨最近一季 EPS 的公布日 | 同上 |

**實作位置：** `backtest_service.py` 加入基本面欄位時，每個資料點都要 shift 對應天數，用戶無感。

---

### 安全沙箱（Q8-A）

**永不用 `eval()`。** DSL parser 路線：
1. 用戶輸入字串 → `lark` parse → AST
2. AST 走訪 → 只允許白名單函數 / 欄位 / 運算子
3. 編譯成 pandas 表達式 → 對 DataFrame 套用
4. 任何不合白名單的 token 都直接拒絕並回友善訊息

---

### 快取策略

- **5 年 K 線**：24h TTL（已實作 `@ttl_cache`）
- **基本面**：1h TTL（用 `fundamental_cache_service.py`）
- **全池掃描結果**：以 `(strategy_hash + pool_hash + date_range)` 為 key 快取 1h
- **參數最佳化結果**：同上策略

## 階段 P3（第四波 · 進階驗證與風控）

### 11. Walk-Forward Analysis（P3-11）

**目的：** 防止策略過擬合。把時間範圍切成 N 個非重疊窗口，每個窗口 IS（訓練）→ 最佳化參數，OOS（測試）→ 驗證。

**規格：**
- 可選窗口數 3/5/8/10
- 可選 IS 比例 50% / 67% / 75%
- 輸出：每窗口最佳參數 + IS/OOS 績效對照表 + OOS 效率比（< 0.5 = 過擬合警告）
- 整體 OOS 資金曲線（拼接）

### 12. Monte Carlo Simulation（P3-12）

**目的：** 量化最大損失風險。對既有回測的交易記錄做 1000 次隨機重排，呈現報酬分佈。

**規格：**
- 純前端計算（不需額外 API）
- 5th / 25th / 50th / 75th / 95th 百分位資金曲線
- 「最壞情況下最大回撤」分佈直方圖

### 13. 交易分佈分析（P3-13）

**目的：** 深化現有 Stats 分頁。

**規格：**
- P&L 報酬率直方圖（bins=20）
- 最長連勝 / 連敗紀錄
- 月份 P&L 熱力圖（year × month）

## 階段 P6（第七波 · 進階量化評估）

### 20. 策略評分卡（P6-20）

**目的：** 把多維度回測指標收斂成一個直覺的「綜合評分」，讓使用者秒懂策略優劣。

**規格：**
- 純前端，利用現有 `BacktestStats`
- 五個評分維度（權重）：
  | 維度 | 滿分 | 評分標準 |
  |------|------|---------|
  | 年化報酬（CAGR）| 25 | ≥30%=25 / ≥20%=20 / ≥15%=15 / ≥10%=10 / ≥5%=5 |
  | 風險調整（Sharpe）| 25 | ≥2=25 / ≥1.5=20 / ≥1=15 / ≥0.5=10 / ≥0=5 |
  | 最大回撤（MDD）| 25 | ≤5%=25 / ≤10%=20 / ≤20%=15 / ≤30%=10 / ≤40%=5 |
  | 勝率（Win Rate）| 15 | ≥70%=15 / ≥60%=12 / ≥55%=9 / ≥50%=6 / ≥40%=3 |
  | 盈虧比（Profit Factor）| 10 | ≥3=10 / ≥2=8 / ≥1.5=6 / ≥1.2=4 / ≥1=2 |
- 等級：A（≥80）/ B（≥65）/ C（≥50）/ D（≥35）/ F（<35）
- 顯示：Stats Tab 頂部彩色評分卡，各維度顯示分條 + 圓圈總分

### 21. 最佳停損 / 停利推薦（P6-21）

**目的：** 用歷史交易的 MAE（最大不利偏移）和 MFE（最大有利偏移）統計，自動推薦最佳停損/停利百分比。

**規格：**
- 後端 `POST /backtest/stop-recommendation`：接收 `trades[]`，以 entry_price 基準計算每筆交易的最壞/最好情況
- 統計最佳截斷點：讓「總盈虧最大化」的 stop_loss / take_profit 組合
- 前端 Stats Tab 底部新增「🎯 停損/停利建議」卡片，顯示推薦值與預估改善幅度
- 若未提供實際 MAE/MFE 資料，退化為用 entry/exit 的 pnl_pct 分佈估算

### 22. 資金曲線趨勢分析（P6-22）

**目的：** 用線性回歸揭露資金曲線長期趨勢，識別「上升通道」或「衰退跡象」。

**規格：**
- 純前端，在「資金曲線」tab（chart）疊加：
  - 線性回歸趨勢線（OLS）
