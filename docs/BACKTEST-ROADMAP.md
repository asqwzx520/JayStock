## 關鍵技術原則

### 基本面 Lookahead 防護（Q15-B）

**為什麼重要：** 2024 Q1 EPS 是 2024-05-15 才公布，回測時若 2024-01-01 就用 EPS=8 來判斷，等於用了「未來資訊」，績效嚴重失真。

**保守延後規則（按法定公布期限）：**

| 資料類型 | 延後天數 | 範例 |
|---------|:------:|------|
| 季 EPS | **+45 天** | Q1 結束 3/31 → 5/15 起可用 |
| 月營收 | **+10 天** | 1 月 → 2/10 起可用 |
| 年營收 | **+90 天** | 年底 12/31 → 隔年 3/31 起可用 |
| TTM EPS | 跟隨最近一季 EPS 的公布日 | 同上 |

**實作位置：** `backtest_service.py` 加入基本面欄位時，每個資料點都要 shift 對應天數，用戶無感。

---

### 安全沙箱（Q8-A）

**永不用 `eval()`。** DSL parser 路線：
1. 用戶輸入字串 → `lark` parse → AST
2. AST 走訪 → 只允許白名單函數 / 欄位 / 運算子
3. 編譯成 pandas 表達式 → 對 DataFrame 套用
4. 任何不合白名單的 token 都直接拒絕並回友善訊息

---

### 快取策略

- **5 年 K 線**：24h TTL（已實作 `@ttl_cache`）
- **基本面**：1h TTL（用 `fundamental_cache_service.py`）
- **全池掃描結果**：以 `(strategy_hash + pool_hash + date_range)` 為 key 快取 1h
- **參數最佳化結果**：同上策略

## 階段 P3（第四波 · 進階驗證與風控）

### 11. Walk-Forward Analysis（P3-11）

**目的：** 防止策略過擬合。把時間範圍切成 N 個非重疊窗口，每個窗口 IS（訓練）→ 最佳化參數，OOS（測試）→ 驗證。

**規格：**
- 可選窗口數 3/5/8/10
- 可選 IS 比例 50% / 67% / 75%
- 輸出：每窗口最佳參數 + IS/OOS 績效對照表 + OOS 效率比（< 0.5 = 過擬合警告）
- 整體 OOS 資金曲線（拼接）

### 12. Monte Carlo Simulation（P3-12）

**目的：** 量化最大損失風險。對既有回測的交易記錄做 1000 次隨機重排，呈現報酬分佈。

**規格：**
- 純前端計算（不需額外 API）
- 5th / 25th / 50th / 75th / 95th 百分位資金曲線
- 「最壞情況下最大回撤」分佈直方圖

### 13. 交易分佈分析（P3-13）

**目的：** 深化現有 Stats 分頁。

**規格：**
- P&L 報酬率直方圖（bins=20）
- 最長連勝 / 連敗紀錄
- 月份 P&L 熱力圖（year × month）

## 階段 P9（第十波 · 可信度 — 成交模型升級）

> **決策背景（grill-me 2026-06-12）：** 現有引擎在訊號產生當根K棒以收盤價成交，存在前視偏差（look-ahead bias）。P9 改成「隔日開盤成交」標準做法，同步升級停損/停利觸發邏輯，確保回測數字更接近實盤。

### 29. 隔日開盤成交模型（P9-29）

**目的：** 消除前視偏差 — 訊號在第 T 日產生，第 T+1 日開盤成交。

**規格：**
- 後端 `backtest_service.py`：引擎迴圈改用 `next_open` 欄位成交
- 買入：以 `next_open * (1 + slippage + buy_cost)` 建倉
- 賣出訊號：以 `next_open * (1 - slippage - sell_cost)` 平倉
- 新增 `slippage_pct` 參數（前端可設定，預設 0.1%）
- 跳空處理：若停損/停利觸發但開盤已跨越停損價 → 以開盤價成交（吃滿跳空損失）
- 前端 `StrategyConfig` 加「滑價 %」輸入欄（0.0 ～ 0.5%）

**影響：** 所有舊歷史快照的數字不可與新版直接比較 → P9-31 引擎版號處理。

### 30. 盤中停損/停利觸發（P9-30）

**目的：** 用當日 high/low 觸發，比收盤觸發更接近實盤。

**規格：**
- 停損觸發條件：`low_price <= entry_price * (1 - stop_loss_pct)`
  - 盤中觸及 → 以停損價成交（`exit_reason: "stop_loss"`）
  - 開盤跳空跨越 → 以開盤價成交（`exit_reason: "stop_loss_gap"`）
- 停利觸發條件：`high_price >= entry_price * (1 + take_profit_pct)`
  - 以停利價成交（`exit_reason: "take_profit"`）
- 需要 K 棒有 open/high/low/close 四欄（yfinance `auto_adjust=True` 已包含）

### 31. 引擎版號（P9-31）

**目的：** 避免不同引擎版本的歷史快照混用誤判。

**規格：**
- 純前端，`BacktestSnapshot` interface 加 `engine_version: number`（舊無此欄為 v1，新引擎寫 v2）
- `saveHistory()` 寫入時帶 `engine_version: 2`
- 「對比前次」若版本不同，顯示 ⚠️ `成交模型不同，數字不可直接比較`
